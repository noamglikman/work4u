# Work4U Technical System Documentation

This document describes the Work4U system structure, AWS resources, API
interfaces, data models, and maintenance workflows.

## 1. Architecture

Work4U is a serverless AWS application:

| Area | Location | Responsibility |
| --- | --- | --- |
| Frontend | `frontend/` | React + Vite + TypeScript web app |
| Backend | `backend/lambdas/` | Python Lambda handlers |
| Shared backend code | `backend/shared/` | Auth helpers and response helpers |
| IaC | `infrastructure/template.yaml` | Learner Lab SAM/CloudFormation stack |
| Production IaC | `infrastructure/template-prod.yaml` | Template that creates IAM policies |
| API contract | `openapi/work4u-api.openapi.yaml` | OpenAPI/Swagger definition |
| Scripts | `scripts/` | Deploy, seed, build, cleanup, package |

Main AWS services:

- API Gateway exposes REST routes.
- Cognito authenticates users and provides JWT claims.
- Lambda implements the backend services.
- DynamoDB stores venues, ratings, preferences, and learning profiles.
- S3 stores venue images and can host the static frontend.

## 3. Auth and Response Format

All deployed API routes are protected by the Cognito authorizer in
`infrastructure/template.yaml`.

Requests use:

```http
Authorization: Bearer <cognito-id-token>
Content-Type: application/json
```

User extraction and admin checks are in `backend/shared/auth.py`.

- Regular users: Cognito `Users` group.
- Admin users: Cognito `Admins` group.
- Admin-only endpoints return `403 FORBIDDEN` for non-admin users.

All Lambda responses use `backend/shared/response.py`.

Success:

```json
{ "success": true, "ok": true, "message": "...", "data": {} }
```

Failure:

```json
{ "success": false, "ok": false, "message": "...", "errorCode": "VALIDATION_ERROR" }
```

Common status codes: `200`, `201`, `400`, `401`, `403`, `404`, `405`, `409`,
`500`.

Common error codes: `UNAUTHORIZED`, `FORBIDDEN`, `VALIDATION_ERROR`,
`NOT_FOUND`, `DUPLICATE_RESOURCE`, `METHOD_NOT_ALLOWED`, `SERVER_ERROR`.

## 5. API Interfaces

### 5.1 `GET /venues`

Lambda: `backend/lambdas/venues/app.py`  
Frontend: `venues.list()`

Purpose: list active venues, with optional filtering and ranking.

Query parameters:

| Name | Type | Required | Notes |
| --- | --- | --- | --- |
| `lat` | number | No | Latitude; used with `lng` and `radiusKm` |
| `lng` | number | No | Longitude; used with `lat` and `radiusKm` |
| `radiusKm` | number | No | Hard distance filter in kilometers |
| `search` | string | No | Searches names, aliases, address, description, type |
| `priceRange` | string | No | `low`, `medium`, `high`, `any`; ranking signal |
| `wifiQuality` | string | No | `low`, `medium`, `high`; ranking signal |
| `quietEnvironment` | boolean string | No | `true` or `false`; ranking signal |
| `needPowerOutlet` | boolean string | No | `true` or `false`; ranking signal |

Response `data`: `VenueSummary[]`.

Important response fields: `venueId`, `name`, `address`, `latitude`,
`longitude`, `priceRange`, `wifiQuality`, `noiseLevel`, `hasPowerOutlets`,
`averageRating`, `currentCrowdLevel`, `mainImageUrl`, `openingHours`,
`distanceKm`, `matchScore`, `learningScore`.

### 5.2 `GET /venues/{venueId}`

Lambda: `backend/lambdas/venues/app.py`  
Frontend: `venues.get(venueId)`

Purpose: return full details for one active venue.

Path parameters:

| Name | Type | Required |
| --- | --- | --- |
| `venueId` | string | Yes |

Response `data`: `VenueDetail`, which includes `VenueSummary` fields plus
`description`, `imageUrls`, and `forecast`.

Returns `404 NOT_FOUND` when the venue does not exist or is inactive.

### 5.3 `GET /preferences`

Lambda: `backend/lambdas/preferences/app.py`  
Frontend: `preferences.get()`

Purpose: return current user's saved preferences.

Request parameters: none.

Response `data`: `UserPreferences` or `null`.

Fields: `userId`, `email`, `quietEnvironment`, `needPowerOutlet`,
`wifiQuality`, `preferredSeatType`, `priceRange`, `favoriteVenueIds`,
`updatedAt`.

### 5.4 `POST /preferences`

Lambda: `backend/lambdas/preferences/app.py`  
Frontend: `preferences.save(input)`

Purpose: create or replace current user's preferences.

Body:

```json
{
  "quietEnvironment": true,
  "needPowerOutlet": true,
  "wifiQuality": "high",
  "preferredSeatType": "table",
  "priceRange": "medium",
  "favoriteVenueIds": ["venue-001"]
}
```

Validation:

| Field | Type / values |
| --- | --- |
| `quietEnvironment` | boolean |
| `needPowerOutlet` | boolean |
| `wifiQuality` | `low`, `medium`, `high` |
| `preferredSeatType` | `table`, `sofa`, `bar`, `any` |
| `priceRange` | `low`, `medium`, `high`, `any` |
| `favoriteVenueIds` | optional string array, deduplicated and capped at 100 |

Response `data`: saved `UserPreferences`.

### 5.5 `GET /ratings/my`

Lambda: `backend/lambdas/ratings/app.py`  
Frontend: `ratings.listMine()`

Purpose: list ratings created by the current user, newest first.

Response `data`: `Rating[]`.

Rating fields: `ratingId`, `userId`, `userEmail`, `venueId`, `venueName`,
`ratingDate`, `crowdLevel`, `wifiRating`, `noiseRating`, `comment`,
`createdAt`, `updatedAt`.

### 5.6 `POST /ratings`

Lambda: `backend/lambdas/ratings/app.py`  
Frontend: `ratings.submit(input)`

Purpose: create a venue rating and recalculate `averageRating` and
`ratingCount`.

Body:

```json
{
  "venueId": "venue-001",
  "crowdLevel": "reasonable",
  "wifiRating": 5,
  "noiseRating": 4,
  "comment": "Quiet and comfortable"
}
```

Validation:

| Field | Type / values |
| --- | --- |
| `venueId` | string, must reference an active venue |
| `crowdLevel` | `free`, `reasonable`, `crowded` |
| `wifiRating` | integer, 1-5 |
| `noiseRating` | integer, 1-5 |
| `comment` | optional string, max 500 chars |

Business rule: one rating per user, venue, and Israel calendar day. Duplicate
same-day submissions return `409 DUPLICATE_RESOURCE`.

Response `data`: created `Rating`.

### 5.7 `PUT /ratings/{ratingId}`

Lambda: `backend/lambdas/ratings/app.py`  
Frontend: `ratings.update(ratingId, input)`

Purpose: update a rating and recalculate the venue rating summary.

Path: `ratingId` string.

Body:

```json
{
  "crowdLevel": "free",
  "wifiRating": 4,
  "noiseRating": 5,
  "comment": "Updated comment"
}
```

Authorization: users can update their own ratings; admins can update any
rating.

Response `data`: updated `Rating`.

### 5.8 `DELETE /ratings/{ratingId}`

Lambda: `backend/lambdas/ratings/app.py`  
Frontend: `ratings.remove(ratingId)`

Purpose: delete a rating and recalculate the venue rating summary.

Path: `ratingId` string.

Authorization: users can delete their own ratings; admins can delete any
rating.

Response `data`:

```json
{ "ratingId": "rating-..." }
```

### 5.9 `GET /learning`

Lambda: `backend/lambdas/learning/app.py`  
Frontend: `learning.get()`

Purpose: return current user's personalization profile.

Response `data`: `UserLearningProfile`.

Fields: `userId`, `categoryScores`, `searchTermScores`, `totalEvents`,
`createdAt`, `updatedAt`.

Default categories: `cafe`, `coworking`, `library`, `academic`, `community`,
`workspace`.

### 5.10 `POST /learning/event`

Lambda: `backend/lambdas/learning/app.py`  
Frontend: `learning.record(input)`

Purpose: record user behavior and update personalization scores.

Supported bodies:

```json
{ "type": "search", "searchTerm": "quiet cafe" }
```

```json
{ "type": "open_venue", "venueId": "venue-001", "placeType": "cafe" }
```

```json
{ "type": "rating", "venueId": "venue-001", "placeType": "coworking", "rating": 5 }
```

Validation: `type` must be `search`, `open_venue`, or `rating`.

Response `data`: updated `UserLearningProfile`.

### 5.11 `POST /admin/venues`

Lambda: `backend/lambdas/admin/app.py`  
Frontend: `venues.create(input)`  
Auth: admin only.

Purpose: create a venue.

Required body fields: `name`, `address`, `latitude`, `longitude`,
`openingHours`, `priceRange`, `wifiQuality`, `noiseLevel`, `hasPowerOutlets`.

Optional body fields: `description`, `imageUrls`, `categoryLabel`,
`accessNote`, `contactNote`, `website`, `phone`, `email`, `currentCrowdLevel`.

Response `data`: created venue item, including server fields `venueId`,
`averageRating`, `isActive`, `createdBy`, `createdAt`, `updatedAt`.

### 5.12 `PUT /admin/venues/{venueId}`

Lambda: `backend/lambdas/admin/app.py`  
Frontend: `venues.update(venueId, input)`  
Auth: admin only.

Purpose: update editable venue fields.

Path: `venueId` string.

Editable fields: `openingHours`, `priceRange`, `wifiQuality`, `noiseLevel`,
`hasPowerOutlets`, `description`, `imageUrls`, `amenities`, `mainImageUrl`,
`categoryLabel`, `accessNote`, `contactNote`, `website`, `phone`, `email`,
`currentCrowdLevel`.

Non-editable through this route: `name`, `address`, `latitude`, `longitude`,
`placeType`, `source`.

Response `data`: updated venue item.

### 5.13 `DELETE /admin/venues/{venueId}`

Lambda: `backend/lambdas/admin/app.py`  
Frontend: `venues.remove(venueId)`  
Auth: admin only.

Purpose: soft-delete a venue by setting `isActive` to `false`.

Path: `venueId` string.

Response `data`:

```json
{ "venueId": "venue-001", "isActive": false }
```

### 5.14 `POST /admin/venues/{venueId}/images/upload-url`

Lambda: `backend/lambdas/admin/app.py`  
Frontend: `venues.uploadUrl(...)`, `storage.uploadVenuePhoto(...)`  
Auth: admin only.

Purpose: create a 900-second S3 presigned URL for direct image upload.

Path: `venueId` string.

Body:

```json
{ "fileName": "workspace.jpg", "contentType": "image/jpeg" }
```

Validation: `contentType` must start with `image/`.

Response `data`:

```json
{
  "uploadUrl": "https://...",
  "imageUrl": "https://<bucket>.s3.amazonaws.com/venues/<venueId>/<file>",
  "objectKey": "venues/<venueId>/<file>"
}
```

Upload flow: request URL, upload bytes with `PUT uploadUrl`, then save
`imageUrl` on the venue.

## 8. Frontend API Layer

The UI uses service interfaces from `frontend/src/api/services.ts`.

Implementations:

- `frontend/src/api/live.ts`: AWS API Gateway adapter.
- `frontend/src/api/mock/store.ts`: mock adapter for local testing.

Flow:

1. UI calls a service method.
2. `live.ts` maps it to a REST endpoint.
3. `http.ts` attaches the Cognito token and unwraps `{ success, data }`.
4. `frontend/src/lib/mappers.ts` converts API data to view models.

Note: `users.me()` and `/users/me` exist in the frontend/OpenAPI contract, but
the current SAM stack does not deploy that route. The UI currently derives the
signed-in user from Cognito in `frontend/src/api/auth.ts`.

## 11. Maintenance Notes

- `GET /venues` scans the venues table and filters in Lambda. For larger
  datasets, add indexes or a search service.
- `recalculate_venue_rating()` should eventually query `venueId-index` directly
  instead of scanning with a filter.
- Venue deletion is soft delete only.
- S3 upload URLs expire after 900 seconds.
- `infrastructure/template.yaml` uses Learner Lab `LabRole`.
- `infrastructure/template-prod.yaml` is intended for AWS accounts where the
  stack can create IAM permissions.
